# AgentBean 简单 Demo 功能需求（Agent 频道版）

## 问题陈述

AgentBean 的完整产品愿景较大，包含协作空间、Agent、频道、任务、记忆、安全和企业能力。为了尽快验证核心体验，本 demo 先只保留一个更小的可演示闭环：用户能看到可用 Agent 池，点开 Agent 查看详情，并创建一个频道选择若干真实 Agent 加入对话。

这个 demo 不追求完整协作平台能力，而是验证 AgentBean 最关键的人机协作体验：人类可以像组建一个临时项目频道一样，从 Agent 池里挑选合适的 Agent，让它们进入频道自我介绍，并在同一个频道中继续交流。

## Demo 目标与成功标准

| 目标编号 | 目标 | 成功标准 |
| --- | --- | --- |
| G-1 | 用户能进入 Agent 协作页面 | demo 默认不要求登录；如工程实现必须登录，则仅允许指定测试账号进入 |
| G-2 | 用户能查看 Agent 池 | 至少 1 个真实 Agent 可显示在 agents 页面中 |
| G-3 | 用户能点开 Agent 详情 | 点击 Agent 后可查看名称、角色摘要、状态、最近活跃时间和连接错误摘要 |
| G-4 | 用户能创建频道并选择 Agent | 创建频道时可从 Agent 池中选择至少 1 个 Agent 加入频道 |
| G-5 | 被选 Agent 能在频道中自我介绍 | 频道创建后，被选 Agent 应在频道中发送自我介绍 |
| G-6 | 用户能在频道中继续交流 | 用户发送消息后，频道内在线 Agent 能收到并回复到同一频道 |
| G-7 | 用户能理解基础异常 | Agent 离线、Agent 处理失败、消息发送失败时，界面应给出明确提示 |

## 范围

### 必须包含

- Agent 协作页面访问；demo 默认不做登录，若工程必须有登录态，则使用指定测试账号。
- 左侧导航包含 `agents` 入口和 `频道` 入口。
- `agents` 页面展示 Agent 池，包含所有可用于 demo 的真实 Agent。
- Agent 卡片或条目可点击，点击后进入 Agent 详情页。
- Agent 详情页展示 Agent 名称、角色摘要、在线状态、最近活跃时间、连接错误摘要和接入命令。
- Agent 必须是真实 Agent，例如 OpenClaw、Codex CLI 等实际 Agent，不使用规则回复、静态假数据或手动切换在线状态作为最终演示口径。
- Agent 在线状态由 Agent 客户端自身连接和心跳驱动。
- `频道` 页面支持创建一个 demo 频道。
- 创建频道时可从 Agent 池中选择若干 Agent 加入频道。
- 频道创建后，被选 Agent 应在频道中发送自我介绍。
- 频道内支持人类用户发送文本消息。
- 频道内在线 Agent 能接收消息，并在同一频道中返回文本回复。
- Agent 断开或心跳超时后，其状态应变为离线或不可用。

### 可选包含

- Agent 最近活跃时间。
- 简单的系统消息，例如“Agent 已上线”“Agent 已离线”“Agent 已加入频道”“Agent 处理失败”。
- 发送失败后的重试按钮。
- 频道名称输入；若不做输入，可固定为 `频道 1`。

### 已确认约束

- 本版 demo 只关注 Agent 维度，不包含机器维度。
- Agent 必须是真实 Agent，例如 OpenClaw、Codex CLI 等。
- Agent 心跳超时时间为 30 秒。
- 频道创建时需要显式选择 Agent，不能默认让所有 Agent 加入所有频道。
- 界面需要展示 Agent 接入命令。
- demo 默认不做登录；如工程实现必须登录，则只使用指定测试账号。

## 非目标

- 本 demo 不包含多协作空间。
- 本 demo 不包含完整频道管理；只要求能创建或进入一个 demo 频道。
- 本 demo 不包含邀请成员、团队成员管理、RBAC 或组织权限体系。
- 本 demo 不包含私信 DM。
- 本 demo 不包含任务认领、任务状态流转或任务看板。
- 本 demo 不包含 Agent 长期记忆、Key Knowledge 编辑或记忆恢复 UI。
- 本 demo 不包含复杂 MCP 工具调用、文件读写、命令执行或代码修改。
- 本 demo 不包含生产级安全沙箱、敏感信息脱敏和完整审计日志。
- 本 demo 不包含移动端、桌面客户端、语音、视频或文件上传。
- 本 demo 不包含商业计费、企业版能力或私有化部署。

## 用户故事

### US-1 进入 Agent 协作页面

作为 demo 用户，我想尽快进入 AgentBean 的 Agent 协作页面，以便直接验证 Agent 选择、频道创建和频道交流能力。

### US-2 查看 Agent 池

作为 demo 用户，我想看到当前可用的真实 Agent 池，以便知道有哪些 Agent 可以被加入频道。

### US-3 查看 Agent 详情

作为 demo 用户，我想点开某个 Agent 查看详情，以便了解它的角色、状态、最近活跃时间和接入情况。

### US-4 创建频道并选择 Agent

作为 demo 用户，我想在创建频道时选择若干 Agent，以便把合适的 Agent 拉入同一个频道协作。

### US-5 Agent 加入频道后自我介绍

作为 demo 用户，我想被选中的 Agent 在频道中先做自我介绍，以便快速理解每个 Agent 能做什么。

### US-6 在频道中发送消息

作为 demo 用户，我想在频道里发送文本消息，以便和已加入频道的 Agent 进行群聊式协作。

### US-7 Agent 在频道中回复消息

作为 demo 用户，我想频道内在线 Agent 能在同一频道中回复我的消息，以便验证人类与 Agent 共享频道的核心体验。

### US-8 处理 Agent 离线或异常

作为 demo 用户，我想在 Agent 离线或处理失败时看到明确提示，以便理解为什么消息无法被处理。

## 验收标准

### US-1 进入 Agent 协作页面

| AC 编号 | 假设 | 当 | 那么 |
| --- | --- | --- | --- |
| AC-1.1 | demo 环境不要求登录 | 用户打开 demo 地址 | 系统应直接进入 Agent 协作页面 |
| AC-1.2 | 工程实现必须存在登录态 | 用户打开 demo 地址 | 系统应仅允许指定测试账号进入 Agent 协作页面 |
| AC-1.3 | 非指定测试账号尝试进入需要登录态的 demo | 系统完成账号校验 | 系统应拒绝访问并提示当前 demo 仅允许指定测试账号 |

### US-2 查看 Agent 池

| AC 编号 | 假设 | 当 | 那么 |
| --- | --- | --- | --- |
| AC-2.1 | 当前已有 Agent 接入 | 用户进入 `agents` 页面 | 系统应以卡片或列表展示 Agent 名称、角色标签和状态 |
| AC-2.2 | 当前没有任何 Agent 接入 | 用户进入 `agents` 页面 | 系统应展示空状态，并提示需要启动或注册 Agent |
| AC-2.3 | 多个 Agent 接入 | 用户进入 `agents` 页面 | 系统应分别展示每个 Agent，不应合并为一个泛化状态 |
| AC-2.4 | Agent 状态变化 | 用户停留在 `agents` 页面 | 页面应根据连接事件或心跳更新 Agent 状态 |

### US-3 查看 Agent 详情

| AC 编号 | 假设 | 当 | 那么 |
| --- | --- | --- | --- |
| AC-3.1 | Agent 卡片已展示 | 用户点击某个 Agent | 系统应进入该 Agent 的详情页或详情面板 |
| AC-3.2 | Agent 详情页打开 | 用户查看详情 | 系统应展示 Agent 名称、角色摘要、状态、最近活跃时间和连接错误摘要 |
| AC-3.3 | Agent 有接入命令 | 用户查看详情 | 系统应展示该 Agent 的接入命令或启动说明 |
| AC-3.4 | Agent 客户端断开或心跳超过 30 秒 | 用户查看详情 | 详情页应显示离线或不可用状态 |

### US-4 创建频道并选择 Agent

| AC 编号 | 假设 | 当 | 那么 |
| --- | --- | --- | --- |
| AC-4.1 | 用户进入 `频道` 入口且还没有频道 | 用户点击创建频道 | 系统应展示创建频道界面 |
| AC-4.2 | 创建频道界面已打开，且 Agent 池中存在 Agent | 用户选择一个或多个 Agent 并提交 | 系统应创建频道，并将被选 Agent 加入该频道 |
| AC-4.3 | 用户未选择任何 Agent | 用户尝试创建频道 | 系统应阻止提交，并提示至少选择 1 个 Agent |
| AC-4.4 | 用户选择了离线 Agent | 用户提交创建频道 | 系统可允许创建，但应标明该 Agent 当前不可用 |
| AC-4.5 | 频道创建成功 | 用户查看左侧 `频道` 列表 | 系统应展示新频道入口，例如 `频道 1` |

### US-5 Agent 加入频道后自我介绍

| AC 编号 | 假设 | 当 | 那么 |
| --- | --- | --- | --- |
| AC-5.1 | 频道创建成功且包含在线 Agent | 用户进入频道 | 每个在线 Agent 应在频道中发送一条自我介绍 |
| AC-5.2 | 被选 Agent 当前离线 | 用户进入频道 | 系统应展示该 Agent 离线提示，不要求其发送自我介绍 |
| AC-5.3 | Agent 自我介绍生成失败或超时 | 系统收到失败结果或超时 | 频道应展示该 Agent 自我介绍失败提示 |
| AC-5.4 | 多个 Agent 同时自我介绍 | 系统展示频道消息流 | 消息流应按服务端接收顺序展示，并明确标识发送者 |

### US-6 在频道中发送消息

| AC 编号 | 假设 | 当 | 那么 |
| --- | --- | --- | --- |
| AC-6.1 | 用户已进入频道 | 用户输入文本并发送 | 系统应将消息展示在频道消息流中 |
| AC-6.2 | 用户发送空消息或仅包含空白字符 | 用户点击发送 | 系统应阻止发送，并保持输入框可编辑 |
| AC-6.3 | 消息发送过程中网络异常 | 用户发送消息 | 系统应展示发送失败状态，并允许用户重试 |
| AC-6.4 | 频道内存在多个在线 Agent | 用户发送消息 | 系统应让频道内具备响应条件的 Agent 接收该消息，具体响应策略可由实现决定 |

### US-7 Agent 在频道中回复消息

| AC 编号 | 假设 | 当 | 那么 |
| --- | --- | --- | --- |
| AC-7.1 | 至少一个频道内 Agent 在线并可处理频道消息 | 用户在频道中发送消息 | Agent 应收到消息并在同一频道中回复 |
| AC-7.2 | Agent 回复消息 | 系统展示频道消息流 | 回复应明确标识发送者为对应 Agent，而不是人类用户 |
| AC-7.3 | Agent 处理失败 | 系统收到失败结果或超时 | 系统应展示 Agent 失败提示或系统消息 |
| AC-7.4 | Agent 在处理过程中离线 | 系统检测到断开 | 系统应停止等待或标记回复失败，并更新 Agent 状态 |
| AC-7.5 | Agent 未加入当前频道 | 用户在频道中发送消息 | 该 Agent 不应接收或回复当前频道消息 |

### US-8 处理 Agent 离线或异常

| AC 编号 | 假设 | 当 | 那么 |
| --- | --- | --- | --- |
| AC-8.1 | 当前频道内所有 Agent 都离线 | 用户进入频道 | 系统应提示当前频道没有在线 Agent，消息可能无人处理 |
| AC-8.2 | 某个 Agent 离线 | 用户查看 Agent 池或频道成员 | 该 Agent 应显示离线或不可用 |
| AC-8.3 | 离线 Agent 重新连接 | 系统收到连接和心跳 | 该 Agent 应恢复在线 |
| AC-8.4 | 用户在当前频道没有在线 Agent 时发送消息 | 系统判断无可用 Agent | 系统应保留人类消息，但提示当前没有 Agent 可响应 |

## 交互规格

| 项目 | 规格 |
| --- | --- |
| 登录入口 | demo 默认不提供登录入口；如工程实现必须登录，则只允许指定测试账号进入 |
| 页面布局 | 左侧为导航栏，包含 `agents` 和 `频道`；右侧为当前视图内容 |
| `agents` 页面 | 展示 Agent 池，Agent 以卡片或列表形式排列 |
| Agent 卡片 | 展示 Agent 名称、角色标签和状态；点击后进入 Agent 详情页或详情面板 |
| Agent 详情 | 展示 Agent 名称、角色摘要、状态、最近活跃时间、连接错误摘要和接入命令 |
| Agent 状态 | 连接中、在线、处理中、离线、异常或未知 |
| `频道` 页面 | 展示频道列表；demo 至少支持创建并进入一个频道 |
| 创建频道 | 创建时从 Agent 池中选择一个或多个 Agent；未选择 Agent 不允许创建 |
| 频道成员 | 频道只包含创建时被选中的 Agent |
| 自我介绍 | 频道创建后，每个在线 Agent 在频道中发送一条自我介绍 |
| 消息展示 | 每条消息展示发送者、发送者类型、人类或 Agent、时间、正文和发送状态 |
| 消息输入 | 频道底部提供文本输入框和发送按钮；空消息不可发送 |
| 系统消息 | Agent 上线、Agent 离线、Agent 加入频道、Agent 自我介绍失败、Agent 处理失败可作为系统消息进入频道 |
| 状态刷新 | Agent 状态应由连接事件或心跳驱动更新；不提供手动切换在线按钮 |

## 边界情况

| 场景类型 | 场景 | 预期行为 |
| --- | --- | --- |
| 空状态 | 无 Agent 接入 | `agents` 页面展示空状态，不展示伪造 Agent |
| 空状态 | 无频道 | `频道` 页面展示创建频道入口 |
| 空状态 | 频道无消息 | 展示开始对话提示；若刚创建频道，应等待在线 Agent 自我介绍 |
| 异常状态 | 必须登录时指定测试账号校验失败 | 展示拒绝访问提示 |
| 异常状态 | Agent 客户端认证失败 | 对应 Agent 显示异常 |
| 异常状态 | Agent 心跳超时 | Agent 变为离线或不可用 |
| 异常状态 | Agent 自我介绍失败 | 频道展示自我介绍失败提示 |
| 异常状态 | Agent 回复超时 | 频道展示处理失败或超时提示 |
| 权限边界 | 非指定测试账号访问需要登录态的 Agent 协作页面 | 拒绝访问并提示使用指定测试账号 |
| 权限边界 | Agent 客户端使用无效凭证连接 | 拒绝连接并记录错误 |
| 权限边界 | Agent 未加入当前频道 | 不接收当前频道消息，也不在当前频道回复 |
| 并发场景 | 多个 Agent 同时连接 | Agent 池应同时展示多个 Agent |
| 并发场景 | 同一 Agent 重复连接 | 系统应按同一 Agent 标识合并或明确展示重复连接，策略需保持一致 |
| 并发场景 | 多个 Agent 同时自我介绍或回复 | 消息流应按服务端接收顺序展示，并保留各自发送者标识 |
| 并发场景 | 用户发送消息时 Agent 刚好离线 | 系统应展示消息已发送，但提示 Agent 可能无法响应 |

## 技术考量

- Agent 在线状态应由真实连接和心跳驱动，不应由前端手动切换。
- demo 可以使用一个固定的 Agent 协作页面，减少协作空间创建、登录和邀请流程。
- Agent 客户端连接服务端时，应携带可识别 Agent 和 Agent 协作页面的凭证。
- 创建频道时需要保存频道与所选 Agent 的成员关系。
- 频道消息应经过服务端广播给当前频道内的在线 Agent，而不是广播给所有 Agent。
- Agent 必须接入真实 Agent，例如 OpenClaw、Codex CLI 等，不使用规则回复或假 Agent 作为最终 demo 口径。
- 当前 demo 不要求 Agent 执行本地命令、读取文件或调用复杂 MCP 工具。
- Agent 心跳超时时间为 30 秒。
- 需要记录最小日志：Agent 连接、Agent 断开、频道创建、Agent 加入频道、自我介绍失败、消息发送、Agent 回复失败。

| D 编号 | 决策 | 影响范围 | 说明 |
| --- | --- | --- | --- |
| D-1 | 本版 demo 只关注 Agent 维度 | 产品、前端、后端 | 页面和需求围绕 Agent 池、Agent 详情、频道创建和频道交流展开 |
| D-2 | Agent 必须是真实 Agent，例如 OpenClaw、Codex CLI | Agent 实现、演示可信度 | 不使用规则回复或假 Agent 作为最终 demo 口径 |
| D-3 | Agent 心跳超时时间为 30 秒 | 后端、前端状态、测试 | 超过 30 秒未收到心跳即视为离线或不可用 |
| D-4 | 创建频道时需要从 Agent 池中选择 Agent | 产品、前端、后端 | 频道只包含被选中的 Agent，不默认包含所有 Agent |
| D-5 | 被选 Agent 加入频道后需要自我介绍 | Agent 行为、频道体验 | 在线 Agent 应发送一条自我介绍；离线 Agent 展示不可用 |
| D-6 | 界面需要展示 Agent 接入命令 | 前端、开发者体验 | 便于演示人员启动真实 Agent 客户端 |
| D-7 | demo 默认不做登录；如必须登录，仅使用指定测试账号 | 登录、测试、演示 | 避免为 demo 增加账号体系复杂度 |
