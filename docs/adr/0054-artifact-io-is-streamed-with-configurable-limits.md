# 文件传输使用流式 I/O 和可配置容量限制

AgentBean 的文件上传、预览和下载必须使用流式 I/O，不能继续把完整文件读入 Server 内存。视频和音频读取支持 HTTP Range，以便按需播放和拖动；预览衍生任务另设解码、像素、时间和资源上限，恶意或异常媒体只能导致预览降级，不能阻断原文件归档。

首个基线默认限制为单文件 250 MB、单次 Agent Run 合计 1 GB，并允许部署配置调整。浏览器在上传前展示限制；Agent Run 超限时保留可见诊断和被跳过文件清单，不能静默遗漏。容量限制是服务保护边界，不用于判断 Artifact role、最终版或文件价值。

Markdown 在线处理采用更严格的默认边界：不超过 2 MB 的 UTF-8 内容可完整预览和编辑；2–10 MB 只提供截断安全预览与下载；超过 10 MB 不进入 Markdown 解析器。每个 revision 最多解析 500 个相对资源引用，超限、解码失败或非 UTF-8 内容必须显示诊断并保持可下载，不能拖垮编辑器或预览服务。
