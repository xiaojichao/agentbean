# PI 治理分为系统、Team 所有者与管理员三层

系统管理员通过 System PI Governance Boundary 控制 Provider 供给、全局 Active PI Model、安全、数据处理和紧急停用；Team 所有者在其内设置 Team PI Governance Ceiling，包括 Server Context、私有内容、本地 Memory 投影、最高 Phase 和可用 Device；Team 管理员只能在该上限内下调 Phase、运营 Agent 分配与 Memory，不能扩大数据边界。Team Owner/Admin 都可以切换独立的“PI 自动协调”总开关。任何 Team 角色都不能查看、选择或覆盖底层模型；货币成本治理不进入 MVP。
