# Team 选择 PI Runtime Profile 而非 Provider 细节

系统管理员负责 PI Provider Supply，包括 endpoint、协议、Credential、模型目录、能力、健康检查、主模型、fallback 和全局成本边界，并发布少量面向质量、速度、成本或本地隐私目标的 PI Runtime Profile。Team 所有者和管理员只从已批准的 Profile 中选择，并继续控制 Phase、自动化、placement 与 Team 预算；Team 不填写 API Key、Base URL、任意 Provider 或模型 ID。实际解析的 Provider、模型、fallback、用量和选择原因仍在 Run 详情中可审计。
