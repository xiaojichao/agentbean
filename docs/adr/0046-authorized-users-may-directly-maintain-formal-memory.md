# 授权用户可以直接维护 Formal Memory

Formal Memory 不只能由 PI Candidate 转换而来。Team Owner/Admin 可以直接创建和编辑 Team Memory 与 Channel Memory；Agent 所有者可以维护自己向 Team 暴露的 Agent Memory 投影，由 Team Owner/Admin 决定是否启用；用户可以直接维护自己的 User Memory。PI 推断内容仍必须先进入 Candidate 队列，经审批后才正式生效。

所有人工创建与编辑都生成新版本，并记录操作者、时间、来源和变更原因。编辑或停用 Memory 不反向修改原频道消息或历史交付；停用后 Memory 立即退出检索和 Active Memory Context，但保留必要的非内容审计事实。Candidate 是降低维护成本的辅助入口，不是 Formal Memory 的唯一入口。
