# 产物路径不决定交付角色

AgentBean 为每个 Agent 运行产物保留 Artifact source root 和根内相对路径，用于发现、隔离与目录导航。Agent 工作目录、每次运行的 `AGENTBEAN_OUTPUT_DIR` 和 Agent 配置的额外输出目录必须保持来源边界，不能把不同根或不同运行中的同名路径自动视为同一文件。

中间产物、普通运行产物和交付物由 Agent 结果清单或 Server 授权流程写入显式 Artifact role；专用目录只能提供默认分类信号，不能仅凭路径、目录名或文件名确定交付物或最终版。最终版继续由人类确认或审核事实建立，以避免目录约定漂移后产生错误项目事实。

频道文件视图只公开每个 source root 的稳定 ID、可读标签和根内相对路径，不公开 Agent 设备上的真实绝对路径。设备所有者或具备设备管理权限的管理员只能在受控诊断面查看真实路径，避免频道文件导航泄露设备用户名、客户目录和本机文件系统结构。

除每次运行固定提供的 `AGENTBEAN_OUTPUT_DIR` 外，Agent 配置必须显式声明额外 source root 的公开名称、环境变量名、默认 Artifact role 和收集策略。Daemon 只在本机解析这些环境变量；系统不能扫描全部环境变量猜测目录。Agent 工作目录扫描继续作为兼容兜底，但只收集本次运行新增或修改的受支持文件，并默认归类为普通 Run artifact。
