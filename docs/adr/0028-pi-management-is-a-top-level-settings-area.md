# PI Management 是独立的设置一级区域

设置侧栏新增与“团队”并列的一级入口“PI Agent”。页面通过明确的系统/Team 作用域切换承载不同权限：系统作用域仅向系统管理员开放 Provider Supply、Active PI Model、Rollout、系统治理边界和运行健康；Team 作用域让有权角色选择 Team，并配置自动化、Memory、Agent Skill coverage、Team 限制和使用情况，不展示或选择底层模型。

系统配置与 Team 配置不出现在同一表单中。Team 详情页不再承载 PI 配置，只保留跳转到相应 Team PI 设置的入口；普通成员只看到理解运行状态和 Task 决策所需的信息。该结构既把 PI 从单个 Team 的附属功能提升为系统底座，又保留 Team 范围的治理入口。
