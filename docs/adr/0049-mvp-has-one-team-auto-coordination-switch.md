# MVP 每个 Team 只有一个 PI 自动协调开关

MVP 不提供自动建 Task、任务分解、定向 Offer、开放认领和 Memory 建议等多个独立开关。每个 Team 只有一个默认开启的“PI 自动协调”总开关，由 Team Owner/Admin 切换。开启时 PI 按已经确认的低风险规则统一执行这些动作；关闭时 PI 仍理解每条频道消息，但只给出建议或等待用户明确要求。

用户显式 `@Agent` 或明确要求创建 Task 不受总开关阻止。高风险、不可逆或 Memory 作用域扩大的动作始终需要确认，不能通过总开关自动授权。该设计保留 MVP 的核心价值，同时避免设置页变成难以理解的功能矩阵。
